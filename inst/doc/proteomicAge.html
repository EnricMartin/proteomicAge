library(proteomicAge)

Overview

proteomicAge computes biological age from SomaScan plasma proteomic data.

Protein columns can be named as gene symbols, UniProt accessions, SomaScan

SL IDs, or SomaScan dot-format SeqIds.

The package includes five conventional proteomic aging clocks:

ClockFunctionMain input format
Tanaka 2018compute_tanaka2018_age()UniProt, gene, SL ID, or SeqId
Lehallier 2019compute_lehallier2019_age()UniProt, gene, SL ID, or SeqId
Sathyan 2020compute_sathyan2020_age()UniProt, gene, SL ID, or SeqId
Oh 2023 conventionalcompute_oh2023_conventional_age()SeqId recommended
Wang 2024 ARIC midlifecompute_wang2024_aric_age()SeqId recommended

It also includes the Global proteomic age ensemble:

compute_global_age()

compute_global_age() implements the Global proteomic clock from Robinson

et al. (2026, Nature Aging). It averages predicted ages and age gaps from the

five conventional clocks.

Input data

Input data should have one row per sample and one column per protein.

ColumnRequiredNotes
Sample IDYesSet with id_col
AgeYesChronological age, set with age_col
SexOptionalSet with sex_col; 0 = male, 1 = female by default
Protein columnsYesNumeric SomaScan RFU values

Supported protein naming conventions:

match_byExample
seqid_dotseq.4374.45, 4374.45, seq.4374.45.2
seqid_slSL003869
geneGDF15
uniprotP36222

Use detect_format() when you are not sure how your protein columns are named.

detect_format(c("seq.4374.45", "seq.3045.72", "seq.11104.130"))
detect_format(c("GDF15", "PTN", "CHI3L1"))
detect_format(c("SL003340", "SL012561"))

Recommended workflow

For a full SomaScan dataset, the usual workflow is:

dat <- read.csv("my_somascan_data.csv")

protein_cols <- setdiff(names(dat), c("SampleID", "Age", "Sex"))
fmt <- detect_format(protein_cols)

global_age <- compute_global_age(
  dat,
  id_col = "SampleID",
  age_col = "Age",
  sex_col = "Sex",
  male_value = 0,
  match_by = fmt
)

The result contains the Global proteomic age and the component clocks:

ColumnMeaning
proteomic_ageMean predicted age across five conventional clocks
age_accelerationMean age gap across five conventional clocks
n_clocksNumber of component clocks used
*_ageComponent clock predicted age
*_age_accelerationComponent clock age gap

Running individual clocks

You can also run the clocks separately.

tanaka_age <- compute_tanaka2018_age(
  dat,
  id_col = "SampleID",
  age_col = "Age",
  match_by = fmt
)

lehallier_age <- compute_lehallier2019_age(
  dat,
  id_col = "SampleID",
  age_col = "Age",
  sex_col = "Sex",
  male_value = 0,
  match_by = fmt
)

sathyan_age <- compute_sathyan2020_age(
  dat,
  id_col = "SampleID",
  age_col = "Age",
  sex_col = "Sex",
  match_by = fmt

oh_age <- compute_oh2023_conventional_age(
  dat,
  id_col = "SampleID",

  match_by = fmt

wang_age <- compute_wang2024_aric_age(

Small executable example

This example uses the Tanaka clock because it has a compact 76-protein model

and can run quickly inside the vignette.

set.seed(123)
prots <- tanaka2018_proteins()

demo <- data.frame(
  SampleID = paste0("P", 1:20),
  Age = round(runif(20, 35, 80), 1),
  Sex = sample(c(0, 1), 20, replace = TRUE),
  stringsAsFactors = FALSE
)

for (sid in prots$seqid_sl) {
  demo[[sid]] <- rlnorm(20, meanlog = log(2000), sdlog = 0.5)
}

tanaka_demo <- compute_tanaka2018_age(
  demo,
  id_col = "SampleID",
  age_col = "Age",
  match_by = "seqid_sl"
)

head(tanaka_demo)

Output interpretation

All clock functions return:

ColumnMeaning
idSample identifier
chronological_ageInput chronological age
proteomic_agePredicted proteomic age
age_accelerationAge gap / residual age acceleration
n_proteins_matchedNumber of clock proteins found in the input
n_proteins_missingNumber of clock proteins not found in the input
match_byProtein identifier convention used

In the current implementation, each individual clock's age_acceleration is

computed as the residual from:

lm(proteomic_age ~ chronological_age)

Positive age acceleration means the sample is predicted to be proteomically

older than expected for its chronological age in that dataset.
