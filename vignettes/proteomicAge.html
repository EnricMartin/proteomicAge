proteomicAge: User Manual & Complete Pipeline

ClawBio

2026-06-18

Overview

proteomicAge computes biological age from plasma proteomic data using published proteomic aging clocks. This vignette covers every function in the package and demonstrates a complete analysis pipeline from raw SOMAscan data to a finished report.

Installation

remotes::install_github("ClawBio/ClawBio", subdir = "proteomicAge")

# Recommended: install SomaScan.db for full protein name mapping
BiocManager::install("SomaScan.db")

compute_tanaka2018_age()

Compute proteomic biological age using the Tanaka et al. (2018) 76-protein clock.

# Create demo data with gene symbols
demo <- data.frame(
  SampleID = paste0("S", 1:5),
  Age      = c(30, 45, 55, 65, 80)
)

# Add a few known proteins
demo[["GDF15"]]   <- c(1500, 3000, 5000, 8000, 15000)
demo[["CHI3L1"]]  <- c(1200, 1800, 2500, 4000, 6000)
demo[["SOST"]]    <- c(800,  1200, 2000, 3500, 5000)
demo[["PTN"]]     <- c(900,  1500, 2200, 3800, 5500)
demo[["ADAMTS5"]] <- c(1000, 2000, 3000, 4500, 7000)

# Compute — uses SomaScan.db to auto-map gene → SeqId
result <- compute_tanaka2018_age(demo, protein_format = "gene")
print(result)

Complete Pipeline

This section walks through a complete analysis: raw SOMAscan data → clean data → proteomic age → report.

Step 1: Load Raw Data

# Simulate a realistic SOMAscan dataset (samples × proteins in RFU)
set.seed(123)
n_samples <- 50
prots <- tanaka2018_proteins()

raw_data <- data.frame(
  SampleID = paste0("P", sprintf("%03d", 1:n_samples)),
  Age = round(runif(n_samples, 25, 85), 1),
  Sex = sample(c(0, 1), n_samples, replace = TRUE)
)

for (i in 1:nrow(prots)) {
  raw_data[[prots$SOMAID[i]]] <- 
    rlnorm(n_samples, meanlog = log(2000), sdlog = 0.5)
}

cat("Loaded:", nrow(raw_data), "samples ×", ncol(raw_data), "columns\n")
## Loaded: 50 samples × 79 columns

Step 2: Validate Input

validate_somascan_input(raw_data)

## Total columns: 79 
## 
## Protein coverage: 100.0% (76/76 required proteins found)
## Overall: VALID
## ==================================

Step 3: Preprocess

clean_data <- preprocess_somascan(raw_data, report_missingness = FALSE)
cat("Preprocessed. Any outliers winsorized, log₂ applied.\n")
## Preprocessed. Any outliers winsorized, log₂ applied.

Step 4: Compute Proteomic Age

Apply both clocks to compare results.

# Tanaka 2018 clock (note: data already log-transformed)
tanaka_age <- compute_tanaka2018_age(clean_data, log_transform = FALSE)
tanaka_age$Age <- clean_data$Age

# Lehallier 2019 clock (on gene-symbol version of the same data)
gene_data <- raw_data
names(gene_data) <- c("SampleID", "Age", "Sex", prots$Gene)
leh_age <- compute_lehallier2019_age(gene_data, protein_format = "gene")
leh_age$Age <- gene_data$Age

cat("Done. Computed proteomic age for", nrow(tanaka_age), "samples.\n")
## Done. Computed proteomic age for 50 samples.

When to use which format

Your data format Use protein_format = Notes
Gene symbols (GDF15) "gene" SomaScan.db maps to SeqId via ALIAS
UniProt IDs (Q99988) "uniprot" SomaScan.db required

Cross-clock compatibility

The same input data (with gene symbols) can be used with both clocks:

data <- data.frame(
  SampleID = "S1",
  Age = 50,
  GDF15  = 5000,
  CHI3L1 = 3000,
  SOST   = 2000,
  PTN    = 4000
)

tanaka_result <- compute_tanaka2018_age(data, protein_format = "gene")
leh_result    <- compute_lehallier2019_age(data, protein_format = "gene")

# Both clocks accept the exact same input format
cat("Tanaka: ", tanaka_result$proteomic_age, "\n")
## Tanaka:  244.5586
cat("Lehallier:", leh_result$proteomic_age, "\n")
## Lehallier: 67.41087

Understanding the Output

Age Acceleration

Age acceleration = residuals of lm(PROage ~ chronological_age).

  • ≈ 0: Biological age matches chronological age for the population average
  • > 0: Predicted older than expected (accelerated aging)
  • < 0: Predicted younger than expected (decelerated aging)

This is NOT simply PROage - Age. Using residuals accounts for the systematic relationship between predicted and chronological age.

When proteins are missing

The n_proteins_matched and n_proteins_missing columns show coverage. Results with many missing proteins should be interpreted cautiously:

if (any(tanaka_age$n_proteins_missing > 0)) {
  cat("Some samples are missing proteins.\n")
  cat("Consider the SomaScan panel version compatibility.\n")
}
